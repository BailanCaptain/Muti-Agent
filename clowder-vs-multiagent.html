📋 一、项目概览

🐾 Clowder AI

定位：全功能 Cat Café 多猫智能体平台，支持多 AI provider（Claude/GPT/Gemini）协作，配备持久化证据库、MCP 工具集、技能系统、实时 WebSocket + HTTP 双通道通信。


TypeScript Fastify 4 Next.js Socket.IO Redis (IORedis) SQLite + sqlite-vec MCP SDK v1 pnpm workspace Hugging Face Biome

📦 packages: api / web / shared / mcp-server

🐱 支持 11+ 猫猫智能体（opus, sonnet, codex, gemini, dare, antigravity...）

💾 Redis (端口 6399) + SQLite evidence.sqlite + 文件 transcript

⚡ Multi-Agent

定位：轻量化多模型控制台，聚焦 Claude + Codex + Gemini 三大 provider 协同工作，基于 SQLite 持久化、WebSocket 实时推送、Callback API 反馈机制，设计更为精简。


TypeScript Fastify 5 Next.js 16 WebSocket (原生) SQLite (node:sqlite) MCP SDK Zustand pnpm workspace Biome

📦 packages: api / shared · app/ (Next.js 16)

🤖 支持 3 大 provider（Claude/范德彪, Codex/黄仁勋, Gemini/桂芬）

💾 仅 SQLite (data/multi-agent.sqlite)，无 Redis 依赖

🔌 三、前后端通信

🐾 Clowder AI — 双通道模式

通道 1：HTTP REST (87 个路由文件)

POST /api/invocations        ← 发起 Agent 调用
GET  /api/messages           ← 历史消息拉取
POST /api/callbacks/*        ← MCP → API 回调
GET  /api/skills             ← Skill Manifest
GET  /api/evidence           ← 证据库搜索
GET  /api/tasks / /api/backlog

通道 2：Socket.IO (带 Room 管理)

// 房间结构
thread:{threadId}     ← 线程隔离广播
user:{userId}         ← 多 Tab 同步
worktree:*            ← Worktree 事件
preview:global        ← 预览广播

// 事件流
agent_message   ← 流式响应块 (text/tool_use/error/done)
join_room       ← 房间加入
cancel_invocation ← 中止运行
💡 Socket.IO Room 机制：同一用户多 Tab 均订阅 user:{userId}，消息实时同步。Room 验证防止越权取消他人调用。

⚡ Multi-Agent — 精简双通道

通道 1：HTTP REST (~8 个路由)

GET  /api/bootstrap          ← 初始化加载
GET  /api/providers          ← Provider 列表
POST /api/session-groups     ← 创建会话组
GET  /api/session-groups/:id ← 获取 Timeline
POST /api/threads/:id/model  ← 切换模型
POST /api/threads/:id/stop   ← 停止调用
POST /api/callbacks/post-message  ← Agent → 公共消息
GET  /api/callbacks/thread-context ← Agent 读上下文

通道 2：原生 WebSocket (广播模式)

// 客户端 → 服务端
{ type: "send_message", threadId, content... }
{ type: "stop_thread", threadId }

// 服务端 → 客户端 (广播所有连接)
{ type: "assistant_delta",   delta }
{ type: "message.created",   message }
{ type: "thread_snapshot",   activeGroup }
{ type: "dispatch.blocked",  attempts[] }
💡 广播模式：所有 WebSocket 连接均收到全量事件，无 Room 隔离。适合小规模单用户场景，多用户时存在信息泄露风险。

🧠 四、记忆处理系统

🐾 Clowder AI — 三层记忆架构

Layer 1: Redis (热存储 · 6399) thread:{id} 线程元数据 message:{tid}:* 消息历史 (append-only) session:* 交付游标、会话状态 task:{id} 任务追踪 invocation:* 调用记录 ↳ 支持 --memory 内存模式降级 Layer 2: SQLite evidence.sqlite (冷存储 · 长期) + sqlite-vec 向量扩展 + Hugging Face transformers 生成 Embedding ┌──────────────────────────────────┐ │ SqliteEvidenceStore │ │ EmbeddingService (HF本地模型) │ │ IndexBuilder (增量索引 30s刷新) │ │ KnowledgeResolver (查询引擎) │ │ VectorStore (余弦相似度检索) │ └──────────────────────────────────┘ 功能: 特征文档/决策日志/Thread摘要/语义搜索 Layer 3: data/transcripts/ (文件存储) TranscriptWriter 记录 session 事件 TranscriptReader 加载封存 session 数据 → Session 启动时 Bootstrap 注入
🔍 摘要压缩：F102 Phase G 通过 Claude 模型对长 Thread 做抽象摘要，配合滚动窗口机制，保持上下文新鲜度。
⏱️ 脏读防抖：IndexBuilder 30s 批量刷新，避免高频写入冲击 SQLite。

⚡ Multi-Agent — 单层 SQLite 存储

仅 SQLite: data/multi-agent.sqlite (WAL 模式) session_groups 协作房间 id, title, created_at, updated_at threads 各 provider 上下文线 id, session_group_id, provider, alias, current_model, native_session_id messages 聊天消息 id, thread_id, role, content, thinking, created_at invocations Agent 执行实例 id, thread_id, agent_id, callback_token, status, started_at, finished_at, exit_code agent_events 执行活动日志 (事件溯源) id, invocation_id, thread_id, agent_id, event_type, payload ⚠ 无向量检索 ⚠ 无摘要压缩 ⚠ 无语义搜索 ⚠ 无持久化会话记忆 (每次仅读当前 thread 历史)
💡 Callback 上下文：Agent 可通过 GET /api/callbacks/thread-context 在执行期间读取 thread 历史，实现有限的上下文感知，但不具备跨 session 记忆能力。
⚠️ 局限：长对话后没有摘要机制，context 窗口满后必须依赖 CLI 的原生 resume 机制，无法主动控制。

🤝 五、A2A Agent-to-Agent 通信

🐾 Clowder AI — 混合 A2A 架构

模式 1：外部 A2A (JSON-RPC 2.0)

// 标准 A2A Protocol 调用
POST https://remote-cat/
Content-Type: application/json
Authorization: Bearer {apiKey}

{
  "jsonrpc": "2.0",
  "id": "",
  "method": "tasks/send",
  "params": {
    "id": "",
    "message": {
      "role": "user",
      "parts": [{"type": "text", "text": "..."}]
    }
  }
}

模式 2：内部 Mention 路由

// 消息中 @mention → AgentRouter 派发
@opus 帮我review这段代码
@gemini 设计这个界面

// 解析规则
- 行首 @catId 或 @alias
- 最长模式优先 (防止 @opus-45 → @opus)
- 剥除代码块后再解析
- 最多 2 个目标/消息
- 深度限制: MAX_A2A_DEPTH=15
🔗 路由策略：有 @mention → 直接路由；有 ideate intent + 多猫 → 并行；有 execute intent → 串行链；默认 → 上一次回复者 / opus。

⚡ Multi-Agent — 内部 Dispatch 机制

DispatchOrchestrator 调度核心

// Mention 解析 (行首匹配)
@范德彪 (Codex)   @黄仁勋 (Claude)
@桂芬 (Gemini)

// 也支持 Markdown 强调格式
**@范德彪** *@桂芬*

// 调度队列
dispatchQueue[sessionGroupId] = []
→ 串行执行 (不并行)
→ 最大 Hop 数: 10
→ 同根消息链防重复 provider

// 防护机制
- hop count ≤ 10
- 同一根消息链不重复触发同 provider
- session 取消时清空队列
⚡ Callback 触发：Agent 执行中可通过 POST /callbacks/post-message 发布公共消息，触发 mention 解析，实现执行中的 A2A 分发。

🔄 A2A 消息流程对比

🐾 Clowder AI — 外部 JSON-RPC

用户消息: "@opus 评审代码" ↓ a2a-mentions.ts → 解析 targets = [opus] ↓ AgentRouter.routeToMentioned() ↓ A2AAgentService.invoke() [如果是远程 cat] ↓ POST tasks/send (JSON-RPC 2.0) → 远程 Cat 实例处理 → 返回 A2ATask 结果 ↓ transformA2ATaskToMessages() ↓ 广播到 thread:{id} room 内部 cat → ClaudeAgentService / GeminiAgentService (走本地 CLI 派发, 非 JSON-RPC)

⚡ Multi-Agent — 内部队列

用户消息: "@范德彪 帮我 review" ↓ DispatchOrchestrator.registerUserRoot() ↓ MentionRouter.parse() → targets = [codex] ↓ dispatchQueue[groupId].push({provider:'codex'}) ↓ MessageService.flushDispatchQueue() ↓ runThreadTurn(codex) → spawn codex CLI ↓ Codex 输出 "@黄仁勋 已完成..." (行首) ↓ Callback POST /post-message DispatchOrchestrator 再次解析 mention ↓ dispatchQueue.push({provider:'claude'}) ↓ flushQueue → runThreadTurn(claude)

⚡ Multi-Agent — 最小化 MCP

MCP 服务仅供 Claude 使用，其他 Provider 用 Callback Prompt 替代：

MCP Server (packages/api/src/mcp/server.ts) Transport: StdioServerTransport 仅 Claude 可用 工具集 (回调封装) post_message → POST /callbacks/post-message [其他工具通过 JSON-RPC 暴露] Codex / Gemini: 不使用 MCP! → 通过 Callback Prompt 注入方式 → Agent 自行调用 HTTP API MCP Config: 内联 JSON 字符串 (不写临时文件，更安全) claude --mcp-config '{ "mcpServers": { "multi-agent": { "command": "node", "args": ["/path/to/mcp-server/dist/index.js"], "env": { "MULTI_AGENT_INVOCATION_ID": "...", "MULTI_AGENT_CALLBACK_TOKEN": "..." } } } }'

Codex / Gemini 的替代方案：

// 环境变量注入给 CLI 子进程
MULTI_AGENT_API_URL=http://localhost:8787

// 自行调用 HTTP callback 端点

🏆 综合结论

🐾 Clowder AI — 企业级多猫平台
适合：需要长期记忆、跨 session 知识沉淀、多 AI provider 协作、复杂工作流自动化的团队。
代价：部署复杂（需 Redis）、代码量庞大、学习曲线陡峭、维护成本高。
核心优势：向量记忆 + 摘要压缩 + 并行 A2A + 11 个 Provider + 20+ Skill + 完整 MCP 工具集
⚡ Multi-Agent — 轻量协作控制台
适合：快速部署、个人/小团队使用、专注 Claude+Codex+Gemini 协作的场景。
代价：无长期记忆、无并行执行、MCP 覆盖有限、Skill 体系弱。
核心优势：零 Redis 部署 + Token 优化 + 明确安全认证 + 新框架 + 代码精简易读
