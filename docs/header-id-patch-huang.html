视图 2Schema 变更 · session_groups 新增两列

UUID 主键不动（零迁移风险）· 新增 roomNumber（递增）+ shortId（UUID 前 8 位冗余）
session_groups {
  id           text     PRIMARY KEY   // UUID v4，主键/FK/API URL
  roomNumber   integer  UNIQUE        // 【新】R-001, R-042 递增编号
  shortId      text     GENERATED     // 【新】substr(id, 1, 8)
  title        text                    // Haiku 自动命名 / 用户重命名
  projectTag   text                    // 降级为卡片胶囊
  namedAt      text                    // 【新】自动命名时间戳；null=未命名
  isEvidence   integer  DEFAULT 0     // 【新】标记为证据（右键菜单）
  createdAt    text
  updatedAt    text
}

-- 生成 roomNumber 的事务（避免并发冲突）
BEGIN;
  SELECT COALESCE(MAX(roomNumber), 0) + 1 AS next_num FROM session_groups;
  INSERT INTO session_groups (id, roomNumber, ...) VALUES (?, ?, ...);
COMMIT;

三层 ID 各司其职

UUID 内部主键、FK、API URL、日志。/api/sessions/5f52c51b-... 这种不对用户暴露
roomNumber 人类可读。UI 上以 R-042 显示；搜索 room:42 直达；跨房间引用粘贴
shortId UUID 前 8 位。中间标题栏小灰字 #5f52c51b；debug、日志对齐、issue 粘贴
title 语义名字。Haiku 在首条 user + assistant 后自动命名；用户可右键重命名锁定
