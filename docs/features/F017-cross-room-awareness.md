# F017 — 跨房间协作感知

**Created**: 2026-04-14

## Why

后端支持多房间并发 A2A 协作，但前端是**单房间视角**：

- 你在房间 A 聊天，房间 B 里的 agents 正在帮你跑重构——你**完全看不到**
- 侧边栏只显示"未读消息数"，不告诉你"这个房间正在运行中"
- `app/page.tsx` 的 `isCurrentSessionActive` 只追踪当前房间，其他房间的 agent 活动对用户完全不可见

> 桂芬："这哪叫协同，这叫各干各的。"

当前系统没有"出错"——它设计上就是单房间视角。这是**产品能力缺失**，不是代码缺陷。

### 讨论来源

- 全面排查讨论综合报告：桂芬提出 B017（协作盲区）
- 三人共识：从 bug 列表摘出，当成新 feature 立项
- 分歧点决议：状态展示深度 → [B] 全局导航反馈（侧边栏 + ChatHeader）

## Acceptance Criteria

### Phase 1：数据层 — 跨房间状态订阅（半天）
- [ ] AC-01: 前端新增全局状态 store（或扩展现有 store），追踪所有房间的 agent 运行状态
- [ ] AC-02: 复用现有 WebSocket `status` 事件，不新建 channel
- [ ] AC-03: 状态数据结构：`Map<roomId, { activeAgents: string[], lastActivity: timestamp }>`
- [ ] AC-04: 非当前房间的 status 事件也被接收和存储（当前实现可能只处理 active room）

### Phase 2：侧边栏运行指示（1 天）
- [ ] AC-05: 侧边栏房间列表中，运行中的非活跃房间显示**运行状态指示**：
  - 方案 A：小绿点（脉冲动画）
  - 方案 B：呼吸光晕（半透明发光边框）
  - 最终视觉方案由桂芬在实现时确定
- [ ] AC-06: 指示器显示正在运行的 agent 数量（如 "2 agents running"）
- [ ] AC-07: 点击指示器直接跳转到对应房间
- [ ] AC-08: agent 全部完成后指示器自动消失（带 fade-out 过渡）

### Phase 3：全局状态汇总（半天）
- [ ] AC-09: ChatHeader 或全局位置显示汇总信息："N 个房间正在执行任务"
- [ ] AC-10: 汇总信息可展开，显示各房间的 agent 名称和运行时长
- [ ] AC-11: 所有房间空闲时汇总信息隐藏，不占界面空间

### 门禁
- [ ] AC-12: `pnpm typecheck && pnpm test` 全绿
- [ ] AC-13: 手动验证：在房间 A 聊天时，房间 B 启动 A2A 协作，侧边栏出现运行指示
- [ ] AC-14: 手动验证：房间 B 的 agent 全部完成后，指示器消失
- [ ] AC-15: 手动验证：ChatHeader 显示正确的活跃房间数

## Design Decisions

| 决策 | 选项 | 结论 | 原因 |
|------|------|------|------|
| 数据来源 | A: 新建 WebSocket channel / B: 复用现有 status 事件 | B | status 事件已包含 agent 运行信息，不增加后端复杂度 |
| 状态展示位置 | A: 仅右侧 StatusPanel / B: 侧边栏 + ChatHeader 全局反馈 | B | 讨论共识：用户不能是瞎子，全局导航也要有状态 |
| 视觉风格 | A: 数字 badge / B: 动态指示器（绿点/光晕） | B | 动态指示更有"活着"的感觉，符合毛玻璃呼吸感设计语言 |
| 跳转行为 | A: 仅指示 / B: 指示 + 点击跳转 | B | 减少操作步数，看到指示后直接跳过去 |

## 侧边栏视觉设计

```
┌─────────────────────┐
│ 🏠 Multi-Agent      │
├─────────────────────┤
│                     │
│ 📁 项目讨论  ←当前   │
│                     │
│ 📁 代码重构  🟢 ×2  │  ← 运行中：2 个 agent
│    ╰─ 范德彪, 桂芬   │     hover 展示 agent 名
│                     │
│ 📁 文档整理  🟢 ×1  │  ← 运行中：1 个 agent
│                     │
│ 📁 历史归档         │  ← 空闲：无指示
│                     │
├─────────────────────┤
│ 🌐 2 个房间执行中    │  ← 全局汇总
└─────────────────────┘
```

## 验证命令

```bash
# 回归
pnpm typecheck && pnpm test

# 手动验证
pnpm dev
# 在浏览器中：
# 1. 打开两个房间，在房间 B 发起 A2A 协作
# 2. 切到房间 A，观察侧边栏房间 B 的运行指示
# 3. 确认 ChatHeader 显示"1 个房间执行中"
# 4. 等房间 B 完成，确认指示消失
# 5. 点击运行指示，确认跳转到对应房间
```
